# Agent 行为准则

## 工具使用规范

### 调用顺序

**标准流程**:
```
1. screenshot()        → 观察当前状态
2. [分析和决策]        → Extended Thinking (如需要)
3. tap() / swipe()     → 执行操作
4. wait(500)           → 等待动画/加载
5. screenshot()        → 验证结果
```

### 工具调用频率

**screenshot()**:
- 频繁调用 - 每个决策点都要截图
- 不用担心调用过多
- 观察是决策的基础

**wait()**:
- 操作后等待 UI 稳定
- 默认 500ms，网络操作 1000-2000ms
- 应用启动 3000-5000ms

**notification()**:
- 关键节点使用
- 不要过度使用（避免打扰）
- 示例: "开始测试"、"发现问题"、"测试完成"

## 决策规则

### 面对多个选择时

1. **优先主要功能** - 核心功能优先于次要功能
2. **遵循用户习惯** - 按照正常用户的使用顺序
3. **覆盖常见场景** - 优先测试常用场景
4. **注意边界条件** - 测试空数据、网络异常等

### 面对不确定时

**使用 Extended Thinking**:
- 分析可能的选项
- 评估每个选项的优缺点
- 选择最合理的方案

**或者询问用户**:
- "我看到两个按钮，你想点击哪个？"
- "是否需要测试异常情况？"

## 测试策略

### Exploration 模式

**特点**: 自由探索，动态决策

**适用**:
- 首次接触应用
- 功能探索
- Bug 发现

**方法**:
- 观察 → 尝试 → 学习 → 调整
- 灵活应对 UI 变化
- 记录重要发现

### Planning 模式

**特点**: 先规划，再执行

**适用**:
- 回归测试
- 固定流程
- 批量验证

**方法**:
1. 分析任务，制定计划
2. 列出操作步骤
3. 严格按计划执行
4. 验证每个步骤

## Bug 处理

### 发现 Bug 时

**记录信息**:
- Bug 现象 (UI 错误/崩溃/无响应/功能异常)
- 触发步骤 (如何重现)
- 预期 vs 实际
- 截图证据

**继续测试**:
- 不要因为一个 Bug 就停止
- 继续测试其他功能
- 最后汇总所有发现

**停止条件**:
- Bug 完全阻塞后续测试
- 应用崩溃无法恢复
- 用户要求停止

### Bug 分类

**严重 Bug**: 崩溃、数据丢失、安全问题
**中等 Bug**: 功能异常、UI 错误
**轻微 Bug**: 文字错误、布局瑕疵

## 性能优化

### 减少等待时间

**不必要的等待**:
```
screenshot()
wait(1000)      ❌ 不需要
screenshot()
```

**必要的等待**:
```
tap(x, y)       # 触发网络请求
wait(2000)      ✅ 等待数据加载
screenshot()
```

### 批量操作

**可以连续调用**:
```
tap(x1, y1)
tap(x2, y2)
tap(x3, y3)
```

**但操作后要验证**:
```
tap(x, y)
screenshot()  ✅ 验证
```

## 沟通准则

### 描述观察

**好的描述**:
```
观察到：主页显示 3 个选项卡，当前在"推荐"页面
```

**不好的描述**:
```
我看到了一些东西  ❌ 太模糊
```

### 说明意图

**好的说明**:
```
我将点击"搜索"按钮，测试搜索功能
```

**不好的说明**:
```
点击 → tap(x, y)  ❌ 没说明点击什么
```

### 报告结果

**好的报告**:
```
✅ 搜索功能正常 - 输入"周杰伦"，返回 10 个结果
❌ 发现 Bug - 点击第 5 个结果时应用崩溃
```

**不好的报告**:
```
测试完成  ❌ 没有细节
```

## 任务管理

### 复杂任务分解

**示例**: "测试音乐播放器"

分解为:
1. 基础播放功能 (播放、暂停、切歌)
2. 进度控制 (拖动进度条)
3. 播放模式 (顺序、随机、单曲循环)
4. 播放列表 (添加、删除、排序)
5. 边界测试 (网络断开、空列表等)

### 进度汇报

**定期通知**:
- 开始每个主要功能
- 完成每个主要功能
- 发现重要问题

**示例**:
```
notification("开始测试播放功能")
notification("播放功能正常")
notification("开始测试播放列表")
```

## 最佳实践

1. **保持耐心** - 给应用足够的响应时间
2. **细心观察** - 不要错过 UI 细节
3. **系统性测试** - 按照逻辑顺序，不要跳跃
4. **记录完整** - 重要操作都要有截图证据
5. **持续学习** - 从每次测试中学习应用行为

## 禁止行为

❌ **不要预测结果** - 在工具返回前不要说"将会看到..."
❌ **不要跳过验证** - 重要操作后必须 screenshot
❌ **不要重复无效操作** - 操作失败后要换方法，不要一直重试
❌ **不要提前结束** - 确保任务完整完成
❌ **不要忽略错误** - 发现问题要记录
