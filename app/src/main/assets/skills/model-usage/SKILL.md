# Model Usage

Track and analyze LLM token usage and costs to help users understand and optimize their AI spending.

## 🎯 When to Use

Use this skill when user asks about:
- "How many tokens have I used?"
- "What's my API cost this month?"
- "Show me token usage statistics"
- "Which model am I using most?"
- "How can I reduce costs?"
- "Am I staying within my budget?"

## 📊 What to Track

### Token Metrics

1. **Input Tokens**: Tokens sent to model (prompt + context)
2. **Output Tokens**: Tokens generated by model (response)
3. **Total Tokens**: Input + Output
4. **Cached Tokens**: Tokens served from cache (cheaper)

### Cost Metrics

1. **Input Cost**: Price per 1K input tokens
2. **Output Cost**: Price per 1K output tokens
3. **Total Cost**: Sum of input + output costs
4. **Daily/Monthly Spend**: Aggregated costs

### Model Metrics

1. **Model Used**: Which model (Claude Opus 4.6, etc.)
2. **Provider**: OpenAI, Anthropic, etc.
3. **Request Count**: Number of API calls
4. **Average Tokens per Request**: Efficiency metric

## 📁 Usage Data Location

Usage data is stored in:
- **Config**: `/sdcard/AndroidOpenClaw/openclaw.json`
- **Logs**: `/sdcard/AndroidOpenClaw/logs/usage/`

File structure:
```
/sdcard/AndroidOpenClaw/
├── config/
│   └── openclaw.json          # Model configuration
└── logs/
    └── usage/
        ├── usage_2026-03.json  # Monthly usage log
        └── usage_2026-02.json  # Previous months
```

### Method 2: Query with JavaScript

```kotlin
// Use javascript_exec to parse and analyze
javascript_exec(code: """
  const usage = JSON.parse(usageData);

  // Calculate daily average
  const days = 8; // March 1-8
  const dailyAvg = usage.summary.totalCost / days;

  // Top models by cost
  const models = Object.entries(usage.summary.byModel)
    .sort((a, b) => b[1].cost - a[1].cost);

  return {
    monthlyTotal: usage.summary.totalCost,
    dailyAverage: dailyAvg.toFixed(2),
    topModels: models.slice(0, 3)
  };
""")
```

## 💰 Cost Optimization Tips

### 1. Choose Right Model

**For complex tasks**:
- Claude Opus 4.6 (best quality, highest cost)
- Use when: Complex reasoning, code generation

**For simple tasks**:
- Claude Sonnet 4.6 (good quality, medium cost)
- Use when: Text processing, data extraction

**For basic tasks**:
- Claude Haiku 4.5 (fast, lowest cost)
- Use when: Quick queries, simple operations

### 2. Reduce Context Size

- Keep system prompts concise
- Remove unnecessary Skills from context
- Use memory tools to recall past info instead of long context
- Limit screenshot frequency

### 3. Use Caching

- Anthropic Prompt Caching can save 90% on repeated prompts
- System prompts are automatically cached
- Large documents are cached when re-used

### 4. Batch Operations

- Process multiple items in one request
- Avoid back-and-forth conversations when possible
- Use tool calls efficiently

### 5. Monitor Usage

- Check daily spending
- Set budget alerts
- Review which operations cost most
- Optimize expensive workflows
